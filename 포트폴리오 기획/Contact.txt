포트폴리오 링크:
https://winter-haeum.github.io/my-portfolio/

== Contact 섹션 UI 컨셉 및 느낌 ==

전체적인 느낌
- 깔끔하고 심플한 느낌
- 친근하고 따뜻한 느낌

설명
- 현재 포트폴리오의 단순하고 깔끔한 구조 유지
- 섹션별 컬러 포인트 스타일 유지
- 초보자 포트폴리오 느낌 유지
- 너무 화려하지 않은 UI
- 여백 중심 레이아웃 사용
- 너무 복잡하지 않은 UI 사용

== 색상 시스템 ==

Contact 섹션 포인트 컬러: 노란 계열
- 섹션 제목 배경: var(--color-contact) = #F5C842
- 버튼 기본 배경: #FFFFFF (흰색, 베이지 배경과 명확히 구분)
- 버튼 활성/선택 상태: #FFD84D (밝은 노랑)
- 버튼 hover: #FFE27A (더 밝은 노랑)
- 내부 그룹 박스 배경: #F8F8F8

공통 버튼 스타일 (네오브루탈리즘):
- border: 2px solid #222
- border-radius: 8px
- box-shadow: 3px 3px 0px #222
- hover 시 shadow 축소 + translate(1px, 1px)

--------------------------------------------------

== 레이아웃 구조 ==

배치 순서 (위 → 아래):
1. Contact 제목 (노랑 배경 박스, 좌측 정렬)
2. 메인 카드 (흰 배경)
   a. 소개 메시지
   b. [구분선]
   c. Email + Links 묶음 박스 (#F8F8F8 배경)
      - EMAIL 라벨 + 이메일 복사 영역
      - LINKS 라벨 + SNS 버튼 3개
   d. [구분선]
   e. 방명록 영역
      - 방명록 작성 폼
      - 방명록 목록 (최신순)

모바일: 세로 정렬 구조 사용

== 이메일 복사 영역 ==

- 이메일 아이콘 + 이메일 주소 텍스트 + 복사 아이콘
- 클릭 시 클립보드에 이메일 주소 복사
- 복사 완료 시 체크 아이콘으로 전환 (2초 후 복귀)
- "이메일이 복사되었어요 ✉️" toast 메시지 자동 표시 후 사라짐
- 흰 배경 + 검정 테두리
- hover 시 그림자 + translate 효과

== 방명록 작성 폼 ==

입력 항목 (순서대로):
1. 오늘의 감정 (이모지 선택)
   - 선택지: 🌱 ☁️ ✨ 💜 🐰 ☕ 🌙
   - 선택: #FFD84D 노랑 배경 + 검정 테두리 + 그림자
   - 미선택: 흰 배경 + 연한 테두리
2. 닉네임 (필수)
3. 소속/직업 (선택, placeholder: 학생, 취준생, 개발자...)
4. 메시지 (필수, 3줄 textarea)
5. 한마디 키워드 (선택)
   - 선택지: 응원 / 성장 / 감성 / 화이팅
   - 선택: #FFD84D 노랑 배경 + 검정 테두리 + 그림자
   - 미선택: 흰 배경 + 연한 테두리
   - 재클릭 시 선택 해제
6. 비공개로 작성하기 체크박스
   - 체크 시: 비공개 글 확인용 비밀번호 입력칸 표시
   - 비밀번호는 선택 입력

※ 이메일 입력칸 없음 (삭제됨)

== 비공개 방명록 기능 ==

작성자 UX:
- 폼 하단 "비공개로 작성하기 🔒" 체크박스 제공
- 체크 시 비밀번호 입력칸 표시
- 비공개 글은 일반 방문자에게 내용 미노출

일반 방문자에게 보이는 화면:
- "비공개 방명록입니다 🔒"
- 작성자 닉네임 + 날짜만 표시

관리자 화면:
- 우하단 "관리자" 텍스트 클릭 → 비밀번호 입력 UI 등장
- 올바른 비밀번호 입력 시 관리자 모드 활성화
- 관리자 모드에서 비공개 글 전체 내용 확인 가능
- 비공개 글에 "🔒 비공개" 배지 표시

== DB 테이블 (portfolio_guestbook) ==

Supabase 프로젝트 ID: slwgtilibpkewwemkccd

컬럼 목록:
1. id            - BIGSERIAL, 자동 생성 (PK)
2. name          - TEXT, 작성자 닉네임 (필수)
3. message       - TEXT, 메시지 내용 (필수)
4. created_at    - TIMESTAMPTZ, 작성 일시 (자동)
5. role          - TEXT, 소속/직업 (선택)
6. emoji         - TEXT, 이모지 선택 (기본값: 🌱)
7. keyword       - TEXT, 한마디 키워드 (선택)
8. is_private    - BOOLEAN, 비공개 여부 (기본값: false)
9. private_password - TEXT, 비공개 글 확인용 비밀번호 (선택)
※ email 컬럼은 DB에 존재하나 UI에서 수집하지 않음

RLS 정책:
- 공개 읽기 허용 (SELECT)
- 공개 쓰기 허용 (INSERT)
